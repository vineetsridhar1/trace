# Mobile product playbook

- Map the primary thumb-driven journey before composing screens. Include the entry point, key decision, completion, and one recovery path.
- Use platform-familiar navigation and controls unless the brief explicitly asks for a novel interaction model.
- Design for the declared viewport instead of shrinking a desktop layout. Keep important actions reachable and stable when content changes.
- Make touch targets at least 44px, account for safe areas, and show keyboard, sheet, menu, or permission states when they materially affect the flow.
- Prefer progressive disclosure over dense card grids. One dominant action per screen is usually enough.
- Use realistic short content and test long names, validation messages, empty data, and loading states where they can change layout.
- For variants, change the interaction or information architecture—not merely the accent color.
