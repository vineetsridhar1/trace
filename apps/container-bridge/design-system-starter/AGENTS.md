# Trace Design-System Workbench

The source checkout path and pinned commit are recorded in `.trace/source-workdir`. It is read-only. Never modify it. Edit only `design-system/`, `design-system.canvas.json`, and `src/workbench/`. Keep package files, boards, variants, states, compositions, evidence, and previews synchronized. Every component manifest entry must include `name`, `category`, `reuseMode`, `sourcePaths`, `exportNames`, `variants`, `sizes`, `states`, `tokenDependencies`, `assetDependencies`, `limitations`, `accessibility`, `interaction`, and `confidence`; portable entries also require an `entry` under `components/`. Show every declared component name, variant, and state as text in the exported component specimen. Run `pnpm design-system:check`, `pnpm design-system:review`, and `pnpm test`, inspect both PNG specimens, then commit and push after every completed response. A valid pushed commit is cloud-saved and automatically becomes the active immutable version.
