# Trace Marketing Video Scenes

This scene outline is built around one continuous visual idea: the same Trace
session persists as the work moves through every context.

Scattered state becomes a Trace session. The session moves from laptop to cloud,
from cloud to mobile, from private work to team collaboration, from one tool to
another, and finally back into the main worktree.

## Scene Outline

| Beat | Voiceover | Visual |
| --- | --- | --- |
| 1. Scattered State | "Everyone is building agent orchestration. But starting another agent was never the hard part." | Fast montage of agent tabs, terminals, branch labels, phone notifications, and teammates asking for status. Everything feels disconnected. |
| 2. Continuity Problem | "The hard part is continuity: keeping AI sessions alive across models, devices, the cloud, and your team." | The scattered elements orbit loosely: model badges, laptop, cloud, phone, teammate cursors. They feel hard to track. |
| 3. Trace Coalesces | "With Trace, the work moves with you." | All scattered state collapses into one clean Trace dashboard. Session cards become stable objects. |
| 4. Laptop Reveal | "Start locally through the desktop bridge..." | Camera zooms out. The Trace dashboard is on a laptop screen. A developer is working locally. A small Desktop Bridge connection lights up. |
| 5. Cloud Move | "...then move the session into the cloud." | A glowing stream of session state rises from the laptop into a clean cloud environment. The session card remains visually identical. |
| 6. Mobile Continuity | "When you step away, the work keeps moving." | The developer closes the laptop, stands, and walks outside. The cloud session keeps pulsing and running behind them. |
| 7. Mobile App | "Follow progress, test your changes, and respond to the agent from the mobile app." | Phone view shows session progress, a preview or test result, and an agent prompt/reply. Consider a quick "Test passed" or mobile browser preview. |
| 8. Private To Shared | "Keep work private until you are ready to share it." | Phone or laptop UI shows the session marked Private. The user taps Share. |
| 9. Team Surface | "Then your team can follow, review, hand off, or resume without losing the thread." | The user meets teammates. The same session appears on each laptop with different roles: one follows activity, one reviews a diff, one resumes the session. |
| 10. Tool Change | "And when the tool changes, the session keeps its context." | Camera pushes back into the shared session UI. The tool/model selector changes while the transcript, files, and context remain anchored. |
| 11. Claude To Codex | "Switch from Claude Code to Codex, or change models mid-session, without starting over." | Claude Code badge switches to Codex, or a model selector changes. The same session timeline continues with no reset. |
| 12. No Handoff Loss | "No copy-paste handoff. No lost context." | Quick old-world contrast: copied text moving into a new blank chat fades away. Return to one continuous Trace timeline. |
| 13. Sync Back | "When the work is ready, sync it back to your main worktree for testing, review, and shipping." | Session output becomes a clean diff and branch state. A sync animation moves changes into the main worktree. Show tests, review, or PR-ready state. |
| 14. Thesis Card | "Same lifecycle. Same context. Same team surface." | Three quick product shots: laptop/cloud, phone, team dashboard. The same session card or ID persists across all three. |
| 15. End Card | "Trace. The control plane for AI coding teams. Full demo coming soon." | Trace logo over a simple product dashboard background. |

## Notes For The Second Half

After the team-sharing scene, keep the story centered on continuity instead of
introducing a new metaphor.

Use the remaining scenes to show:

- Same session, different tool: the session UI switches from Claude Code to Codex while the timeline, files, branch, and context stay visible.
- No copy-paste handoff: contrast the old world of copying context into a blank chat with Trace's continuous session timeline.
- Sync back to repo: show the AI work becoming real engineering work through diffs, tests, branch state, review, and sync to the main worktree.
- Final continuity montage: bring back laptop, cloud, phone, team, tool switch, and worktree, all connected by the same session thread.

Avoid introducing brand-new metaphors after the city/team scene. The whole video
should feel like one object persists through everything:

scattered state -> Trace session -> laptop -> cloud -> phone -> team -> tool
switch -> repo.
