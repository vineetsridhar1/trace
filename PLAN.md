# Plan: Audit Server ŌĆö Remove Local Machine Dependencies

The server will be deployed on AWS and must not access the user's local filesystem or run local processes. Only the Electron app can interact with the user's machine. This plan moves all local-machine-dependent code out of the server.

---

## 1. Update CLAUDE.md ŌĆö Document Server Architecture

Add a new "Server Architecture" section to CLAUDE.md explaining:
- The server is deployed on AWS and cannot access the user's local machine
- Only the Electron main process can run local operations (git, filesystem, child processes)
- The server communicates with clients only via GraphQL/REST API
- Local operations (git, transcript reading, config files) must go through Electron IPC

---

## 2. Remove Git Operations from Server ŌåÆ Move to Electron IPC

**Problem:** `server/src/services/gitService.ts` runs `git` commands via `child_process.execFile` on the user's local repos. Two GraphQL resolvers (`validateRepo`, `repoBranches`) call these functions.

**Server deletions:**
- Delete `server/src/services/gitService.ts`
- Delete `server/src/schema/channel/resolvers/Query/validateRepo.ts`
- Delete `server/src/schema/channel/resolvers/Query/repoBranches.ts`
- Remove `validateRepo` query, `repoBranches` query, and `RepoValidation` type from `server/src/schema/channel/schema.graphql`
- Run `make gql` to regenerate schema types

**Electron additions:**
- Add `validate-repo` IPC handler in `src/main/ipc.ts` ŌĆö runs `git rev-parse --is-inside-work-tree` and `git remote get-url origin` (reusing existing `runProcess` helper)
- Add `list-repo-branches` IPC handler ŌĆö runs `git for-each-ref --format=%(refname:short) refs/heads/`
- Add preload bridge methods `validateRepo(path)` and `listRepoBranches(path)` in `src/preload.ts`

**Frontend changes:**
- Update `src/components/CreateChannelModal.tsx` to call `window.traceAPI.validateRepo()` and `window.traceAPI.listRepoBranches()` via IPC instead of GraphQL
- Remove the GQL query definitions for `ValidateRepo` and `RepoBranches`
- Delete the generated file `src/components/__generated__/CreateChannelModal.generated.ts` (will be regenerated by `make gql` without those queries)

## 3. Remove Local Config Management from Server

**Problem:** `server/src/services/channelService.ts` writes to `~/.trace/local-config.json` and uses `process.cwd()` + `os.homedir()` to resolve repo paths. The Electron app already has `src/main/localConfig.ts` that fully manages this file.

**Server changes to `channelService.ts`:**
- Remove `ensureDefaultChannelLocalConfig()` function entirely
- Remove `fs`, `os`, `path` imports
- Remove `getOriginRemoteUrl` import (from gitService which is being deleted)
- Simplify `getDefaultChannel()`: remove `process.cwd()` path resolution, remove `getOriginRemoteUrl()` call, remove `ensureDefaultChannelLocalConfig()` call. Just find/create the "general" channel in the DB with a default serverId.

**No Electron changes needed** ŌĆö `src/main/localConfig.ts` already fully manages local config, and the frontend already calls IPC for local config in CreateChannelModal.

## 4. Move Transcript Parsing from Server ŌåÆ Electron

**Problem:** `server/src/services/eventService.ts` reads Claude transcript files from disk (`fs.openSync`, `fs.readFileSync`) and runs `execSync('git rev-parse')` to resolve branches. `server/src/services/messageService.ts` also does lazy transcript enrichment on fetch.

### Server changes to `eventService.ts`:
- Remove functions: `extractUsageFromTranscript`, `extractAskUserQuestionFromTranscript`, `extractExitPlanModeFromTranscript`, `resolveGitBranch`
- Remove `fs` and `child_process` imports
- Remove transcript file reading from `ingestEvent()` (the code that calls extract functions using `payload.transcript_path`)
- Remove the delayed retry `setTimeout` blocks (1.5s, 4s retries + 5s auto-complete)
- Instead, accept pre-extracted data from the event payload:
  - `payload.extracted_usage` ŌåÆ `{ input_tokens, output_tokens }`
  - `payload.extracted_tool_name` ŌåÆ `'AskUserQuestion' | 'ExitPlanMode'`
  - `payload.extracted_tool_input` ŌåÆ tool input data (the questions array, etc.)
  - `payload.branch_name` ŌåÆ resolved git branch name
- Use these fields in `ingestEvent` where it currently reads from files
- Keep the auto-complete logic but trigger it based on a simpler timer without transcript re-reads

### Server changes to `messageService.ts`:
- Remove lazy enrichment blocks in `getEventsByMessage()` and `getEventsByThread()` that look up session.transcriptPath and call extract functions
- Remove the `extractAskUserQuestionFromTranscript` and `extractExitPlanModeFromTranscript` imports
- The data will already be on the event from Electron's enriched Stop event

### Server changes to `types/hookEvents.ts`:
- Add optional fields to schemas: `extracted_usage`, `extracted_tool_name`, `extracted_tool_input`, `branch_name`

### Electron additions:
- Create `src/main/transcript.ts` with the three extract functions (moved from server, same logic)
- In `src/main/claude.ts` `child.on('close')` handler:
  - Read the transcript file after the process closes
  - Extract usage, AskUserQuestion, ExitPlanMode data
  - Resolve the git branch via `git rev-parse --abbrev-ref HEAD` in the worktree
  - Include all extracted data in the synthetic Stop event payload
  - Modify logic so enrichment data is ALWAYS sent even if a hook Stop was already received (send a follow-up enrichment event)
  - Add retry logic (1.5s, 4s delays) for transcript reading since the file may not be written immediately
